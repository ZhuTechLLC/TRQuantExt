# Cursor Opus 任务模板 (Task Prompt Template)

在使用 Cursor + Claude Opus 4.5 开发时，可参考以下结构编写提示词，以确保 Claude 理解任务并给出符合预期的输出：

**任务**：*(简要描述需要完成的开发任务或问题。例如："实现一个新的API接口用于获取用户投资组合数据。")*

**背景**：*(提供必要的背景信息，如模块名称、相关代码段、当前实现的局限。例如："当前我们有一个获取单个交易账户信息的API，需要扩展支持返回用户的完整投资组合情况，包括各项资产明细。")*

**要求**：

- *(列出具体要求或验收标准。例如："API需要符合REST风格，路径为`GET /api/portfolio`; 响应包含股票、债券、市值等字段; 必须检查用户权限。")*

- *(继续罗列其他要求，如性能指标、错误处理要求等。)*

**约束**：

- *(列出任何技术或业务上的限制条件。例如："必须复用现有的数据库查询方法，不得引入新的依赖; 考虑大数据量情况下的性能优化。")*

- *(如果没有特殊约束，此部分可留空或省略。)*

**输出**：

*(明确期望Claude输出的内容形式。例如："请给出修改后的代码片段，包括控制器与服务层的实现，以及必要的注释和单元测试。无需重复已有代码，其它文件如无变动可省略。")*

> 提示：确保提示中包含足够的信息和上下文，让 AI 明确知道在哪个模块、哪个功能上进行改动或新增代码。必要时，可以在**背景**部分提供相关文件路径和函数签名，或在Cursor中选中相应代码片段后再提问。清晰的任务描述和约束有助于Claude更准确地完成请求并控制token开销。
