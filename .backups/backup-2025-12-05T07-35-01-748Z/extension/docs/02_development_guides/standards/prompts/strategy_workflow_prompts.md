# 策略回测与报告标准 Prompt 模板

为了在 Cursor / ChatGPT 中复用量化工作流程，本文件汇总了常用的提示词模板。将策略名称、日期范围等参数替换为项目当前的实际值即可。

---

## 1. 初始化与需求确认

```
我们需要针对 {策略名称} 策略进行一次回测，请按照我们的标准流程执行：
1. 确认所需的策略文件、依赖和配置是否齐全；
2. 回顾最新的策略版本记录；
3. 给出执行该回测前需要准备的数据和参数清单；
4. 等我确认后再继续下一步。
```

用途：在正式开始回测前，快速确认上下文、依赖和输入参数。

---

## 2. 回测执行提示

```
请帮我在本地运行 {策略名称} 策略的回测：
- 回测时间：{开始日期} 至 {结束日期}
- 交易标的：{股票代码列表}
- 使用 `python run_{策略标识}.py` 脚本
- 回测完成后，请确认 JSON 与 HTML 报告已经生成，并复制到 `strategies/{策略目录}/reports/`
- 输出关键指标（收益、夏普、最大回撤、交易次数）供我快速浏览
```

用途：执行标准回测并提醒助手同步归档结果文件。

---

## 3. 报告复核与分析

```
读取最新的 `strategies/{策略目录}/reports/` 下的 HTML 报告，按照以下结构整理分析要点：
1. 核心绩效指标（收益、风险、交易活动）
2. 市场环境识别与策略参数动态调整情况
3. 策略亮点与潜在风险
4. 针对高增长行业/股票的表现总结
5. 下一步优化建议（含参数调优、数据补充或新实验）
```

用途：在报告生成后，提炼可复用的复盘结论。

---

## 4. 工作流程更新

```
请把本次策略改动与回测过程纳入到 `docs/project_guides/WORKFLOW_GUIDE.md` 的标准流程里：
- 补充需要新增的步骤或注意事项
- 如有新的目录结构或归档规则，也同步更新 README
- 给出一句总结说明我们为什么要做这次调整
```

用途：确保文档与实际操作保持一致。

---

## 5. 策略迭代规划

```
基于最新回测报告和交易记录，为 {策略名称} 策略提出下一阶段的迭代路线：
- 需要新增或改进的信号/因子
- 需要补充的数据源或加工方式
- 对回测框架的潜在增强
- 预估投入的工作量（轻量/中等/重度）
```

用途：形成下一轮迭代的结构化 TODO。

---

## 6. 常用追问

- “当前策略还有哪些版本？各自的定位是什么？”
- “请列出最近三次回测的时间、参数和报告地址。”
- “请检查 `strategies/{策略目录}/versions/` 是否缺少变更记录或注释。”
- “还有哪些行业或主题需要补充公司简介与投资建议？”

这些追问可以嵌入在任意阶段，用于快速定位问题或补充信息。

---

> 建议：将常用模板保存为 Cursor Snippets，配合 `strategy_name`、`start_date` 等变量使用，可大幅提升策略迭代效率。
