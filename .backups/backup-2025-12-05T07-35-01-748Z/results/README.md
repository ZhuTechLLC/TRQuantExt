# 回测报告使用说明

## 查看报告

### 方式1：直接打开完整报告（推荐）⭐
1. 双击 `report_ma_cross_20251106_235011.html` 文件
2. 这是包含完整数据的独立HTML文件，无需加载外部JSON
3. 可以直接在浏览器中打开，无需HTTP服务器

### 方式2：使用模板文件（需要HTTP服务器）
1. 在项目根目录运行：`python -m http.server 8000`
2. 访问：`http://localhost:8000/results/report_template.html`
3. 如果使用file://协议，页面会自动显示文件选择器

### 方式3：通过报告列表（推荐）⭐
1. 打开 `view_report.html` 查看所有可用报告
2. 报告列表会自动扫描并显示所有JSON和HTML报告文件
3. 点击报告链接即可查看
4. 列表会在每次生成新报告后自动更新

## 报告内容

报告包含以下部分：

1. **📈 回测概览** - 关键指标的可视化展示
2. **💡 策略原理** - 详细的策略说明和原理
3. **💻 代码说明** - 策略代码实现和解释
4. **📊 结果分析** - 收益、风险、交易分析
5. **📉 可视化图表** - 资产变化曲线图

## 报告特性

- ✅ 响应式设计，支持移动设备
- ✅ 美观的图表和数据可视化
- ✅ 完整的策略说明和分析
- ✅ 自动计算和评价
- ✅ 无需Python，纯HTML+JavaScript

## 报告列表自动更新

报告列表页面 (`view_report.html`) 会自动扫描并显示所有可用的回测报告：

- **自动扫描**：自动发现所有 `backtest_*.json` 和 `*_report*.html` 文件
- **详细信息**：显示策略名称、日期范围、收益指标等关键信息
- **自动更新**：每次运行回测或生成报告后，列表会自动更新
- **手动更新**：如需手动更新列表，运行 `python update_report_list.py`

## 注意事项

- 报告使用Chart.js和Plotly.js库（通过CDN加载）
- 需要网络连接才能加载图表库
- 如果JSON文件路径不正确，会显示错误提示
- 缺少 `portfolio_history` 数据的JSON文件会显示警告标记，无法生成完整图表
