# 第六章：交易策略与执行

> 好的选股策略需要配合好的交易执行。本章介绍如何将选股信号转化为实际交易，以及A股交易执行的注意事项。

## 6.1 交易执行概述

### 6.1.1 交易执行的三个层次

- **信号层**：选股信号（买入/卖出哪些股票）、仓位信号（每只股票的目标权重）
- **订单层**：订单类型（限价单/市价单）、订单数量、订单时机
- **执行层**：成交管理、滑点控制、异常处理

### 6.1.2 A股交易执行的特殊性

| 特点 | 影响 | 应对策略 |
|------|------|----------|
| T+1制度 | 当日买入无法卖出 | 谨慎买入决策 |
| 涨跌停板 | 可能无法成交 | 分批建仓、提前布局 |
| 集合竞价 | 开盘价格不确定 | 避免集合竞价下单 |
| 流动性差异 | 小盘股冲击成本高 | 控制单笔订单规模 |

## 6.2 订单类型与策略

### 6.2.1 订单类型

| 订单类型 | 说明 | 适用场景 |
|----------|------|----------|
| 限价单 | 指定价格成交 | 控制成本、不急于成交 |
| 市价单 | 以市场价成交 | 快速成交、流动性好 |
| 对手方最优 | 以对手方最优价成交 | 快速成交、控制滑点 |
| 本方最优 | 以本方最优价成交 | 排队等待、降低成本 |

### 6.2.2 订单策略

**TWAP（时间加权平均价）**：将大单拆分为多个小单，在指定时间内均匀执行

**VWAP（成交量加权平均价）**：根据历史成交量分布，在成交量大的时段多下单

**冰山订单**：只显示部分订单量，成交后自动补充

## 6.3 建仓与减仓策略

### 6.3.1 建仓策略

**一次性建仓**：适用于流动性好、仓位小的情况

**分批建仓**：
- Day 1: 建仓30%
- Day 2: 建仓30%
- Day 3: 建仓40%

**金字塔建仓**：
- 首次建仓: 40%（确认信号）
- 加仓1: 30%（趋势确认）
- 加仓2: 20%（突破确认）
- 加仓3: 10%（最后确认）

### 6.3.2 减仓策略

**目标价减仓**：盈利达到预设目标时分批减仓

**移动止盈**：从最高点回撤一定比例时减仓

**信号减仓**：因子得分下降或跌破关键支撑时减仓

## 6.4 仓位管理

### 6.4.1 仓位计算方法

- **等权重法**：单股仓位 = 总仓位 / 持股数量
- **因子得分加权**：根据因子得分分配权重
- **波动率加权**：波动率低的股票配置更高权重

### 6.4.2 仓位约束

| 约束类型 | 建议值 | 说明 |
|----------|--------|------|
| 总仓位 | 60-90% | 留有现金缓冲 |
| 单股上限 | 5-10% | 控制个股风险 |
| 单股下限 | 1-2% | 确保有意义的配置 |
| 行业上限 | 20-30% | 控制行业风险 |

## 6.5 交易成本控制

### 6.5.1 交易成本构成

| 成本类型 | 费率 | 说明 |
|----------|------|------|
| 佣金 | 0.02-0.03% | 双边收取 |
| 印花税 | 0.1% | 仅卖出收取 |
| 过户费 | 0.001% | 沪市收取 |
| 滑点 | 0.1-0.5% | 因市场冲击产生 |

### 6.5.2 成本控制策略

- 降低换手率：延长调仓周期，设置调仓阈值
- 优化执行时机：避开开盘和收盘，选择成交量大的时段
- 选择合适的券商：比较佣金费率，关注API接口支持

## 6.6 PTrade/QMT交易执行

### 6.6.1 执行框架要点

1. 获取当前持仓
2. 计算目标市值
3. 计算需要调整的金额
4. 先卖出，再买入
5. 处理订单异常

### 6.6.2 异常处理

- 订单被拒绝：记录原因，调整策略
- 部分成交：监控剩余订单
- 订单取消：分析原因，决定是否重新下单

## 6.7 本章小结

交易执行是策略落地的关键环节：

1. **订单策略**：根据流动性选择合适的订单类型
2. **建仓策略**：分批建仓降低冲击成本
3. **仓位管理**：合理分配权重，控制风险
4. **成本控制**：降低换手率，优化执行时机
5. **异常处理**：做好订单监控和异常处理
